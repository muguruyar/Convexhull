Milyonluk veri setleri için NumPy ile optimize edilmiş hibrit (Akl-Toussaint + Monotone Chain) Convex Hull uygulaması
High-speed Convex Hull hybrid (Akl-Toussaint + Monotone Chain). Optimized with NumPy vectorization. 
-

1. Akl-Toussaint Heuristic (Pruning)İlk aşamada, veri setindeki uç noktalar (P_{min\_x}, P_{max\_x}, P_{min\_y}, P_{max\_y}) tespit edilerek bir dış bükey dörtgen oluşturulur. Bu alanın içinde kalan noktalar, dış kabuğun bir parçası olamayacağı için elenir.Vektörizasyon: Eleme işlemi, NumPy kütüphanesi aracılığıyla SIMD (Single Instruction, Multiple Data) prensibiyle gerçekleştirilir. Bu, milyonlarca noktanın eş zamanlı olarak test edilmesine olanak tanır.
2. Andrew's Monotone Chain (Finalization)Ön işleme (pruning) adımından sonra veri boyutu yaklaşık %99.9 oranında azaltılır. Geriye kalan aday noktalar, sıralı ve kararlı bir yapı sunan Monotone Chain algoritmasıyla işlenir.Sıralama Maliyeti: Veri seti dramatik ölçüde küçüldüğü için O(n \log n) olan sıralama maliyeti ihmal edilebilir bir seviyeye çekilmiş olur.

1. Akl-Toussaint Heuristic (Pruning)In the initial stage, the extreme points of the dataset (P_{min\_x}, P_{max\_x}, P_{min\_y}, P_{max\_y}) are identified to construct a convex quadrilateral. Points residing within this area are eliminated, as they cannot mathematically be part of the convex hull.Vectorization: The pruning process is executed using the NumPy library based on SIMD (Single Instruction, Multiple Data) principles. This enables the simultaneous testing of millions of data points, ensuring high computational efficiency.
2. Andrew's Monotone Chain (Finalization)Following the pruning stage, the dataset size is reduced by approximately 99.9%. The remaining candidate points are then processed using the Monotone Chain algorithm, which provides a sorted and stable approach to constructing the final hull.Sorting Overhead: Since the dataset is drastically reduced before this stage, the O(n \log n) sorting cost is minimized to a negligible level, allowing the algorithm to approach near-linear time complexity in practice.
